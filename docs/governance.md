# 문서 거버넌스 가이드

## 📋 목적

프로젝트 내 모든 문서(.md)의 일관성, 가독성, 유지보수성을 보장하기 위한 거버넌스 가이드입니다.

---

## 🗂️ 문서 분류 체계

### 1. 계층 구조

```
프로젝트 루트/
├── README.md                    # 프로젝트 메인 README
├── docs/                        # 문서 디렉토리 (권장)
│   ├── README.md               # 문서 인덱스
│   ├── governance.md            # 이 파일
│   └── {카테고리}/              # 카테고리별 문서
├── {모듈}/                      # 모듈별 문서
│   ├── README.md               # 모듈 README
│   └── {문서명}.md              # 모듈 관련 문서
└── ARCHIVE/                     # 아카이브 문서
    └── {문서명}.md
```

### 2. 문서 카테고리

#### A. 프로젝트 레벨 문서
- **README.md**: 프로젝트 메인 README (필수)
- **ROADMAP.md**: 프로젝트 로드맵
- **CHANGELOG.md**: 변경 이력

#### B. 모듈 레벨 문서
- **{모듈}/README.md**: 모듈 개요 및 사용법
- **{모듈}/DESIGN_*.md**: 설계 문서
- **{모듈}/IMPLEMENTATION_*.md**: 구현 문서
- **{모듈}/COMPLIANCE_*.md**: 검증 문서
- **{모듈}/CONTRACT_*.md**: 계약 문서

#### C. 특수 문서
- **setup_*.md**: 설정 가이드
- **schema.md**: 스키마 정의
- **FORMAT_*.md**: 포맷 명세

## 📝 명명 규칙

### 파일명 규칙

1. **대문자 사용**: 중요 문서는 대문자로 시작
   - 예: `DESIGN_REVIEW.md`, `OUTPUT_CONTRACT.md`

2. **언더스코어 구분**: 단어는 언더스코어로 구분
   - 예: `SECONDARY_FILTER_IMPLEMENTATION.md`

3. **카테고리 접두사**: 문서 유형을 접두사로 표시
   - `DESIGN_*`: 설계 문서
   - `IMPLEMENTATION_*`: 구현 문서
   - `COMPLIANCE_*`: 검증 문서
   - `CONTRACT_*`: 계약 문서
   - `FORMAT_*`: 포맷 명세
   - `AUDIT_*`: 감사 문서

4. **README.md**: 모듈/디렉토리 개요 문서

### 문서 제목 규칙

1. **H1 (#)**: 문서 제목 (파일명과 일치 권장)
2. **H2 (##)**: 주요 섹션
3. **H3 (###)**: 하위 섹션

## 📋 문서 템플릿

### 설계 문서 템플릿

```markdown
# {문서 제목}

## 📋 개요

{문서 목적 및 범위}

## 🎯 목표

{설계 목표}

## 🏗️ 설계

{설계 내용}

## ✅ 검증

{검증 방법 및 결과}

## 📝 참고

{관련 문서 링크}
```

### 구현 문서 템플릿

```markdown
# {구현 제목}

## ✅ 구현 완료 사항

{구현 내용}

## 📋 주요 변경 파일

{변경된 파일 목록}

## 🔄 처리 흐름

{처리 흐름 설명}

## 🧪 테스트

{테스트 방법}
```

### 검증 문서 템플릿

```markdown
# {검증 제목}

## 📋 검증 항목

{검증 항목 목록}

## ✅ 검증 결과

{검증 결과}

## 📊 종합 검증 결과

{종합 결과}
```

## 🔍 문서 품질 기준

### 필수 요소

1. **명확한 제목**: 문서 목적이 명확히 드러나는 제목
2. **개요 섹션**: 문서의 목적과 범위 설명
3. **목차**: 긴 문서는 목차 포함
4. **관련 문서 링크**: 관련 문서 참조

### 권장 요소

1. **예시 코드**: 코드 예시 포함
2. **다이어그램**: 복잡한 구조는 다이어그램 포함
3. **체크리스트**: 검증 항목은 체크리스트 형식
4. **버전 정보**: 문서 버전 및 최종 업데이트 날짜

## 🔄 문서 라이프사이클

### 1. 생성 (Create)
- 새 기능/모듈 추가 시 문서 생성
- 템플릿 사용 권장

### 2. 업데이트 (Update)
- 기능 변경 시 관련 문서 업데이트
- 변경 이력 기록 권장

### 3. 검토 (Review)
- 주요 변경 시 문서 검토
- 정확성 및 일관성 확인

### 4. 아카이브 (Archive)
- 더 이상 사용하지 않는 문서는 `ARCHIVE/`로 이동
- 삭제하지 않고 보관

## 📊 문서 상태 관리

### 문서 상태 표시

문서 상단에 상태 표시:

```markdown
<!-- 문서 상태: draft | review | approved | archived -->
<!-- 최종 업데이트: YYYY-MM-DD -->
<!-- 검토자: {이름} -->
```

### 상태 정의

- **draft**: 초안 (작성 중)
- **review**: 검토 중
- **approved**: 승인됨 (최종)
- **archived**: 아카이브됨

## 🔗 문서 간 연결

### 내부 링크

```markdown
[문서 제목](./path/to/document.md)
[섹션 제목](./path/to/document.md#section-name)
```

### 외부 링크

```markdown
[외부 리소스](https://example.com)
```

### 상대 경로

- 같은 디렉토리: `./document.md`
- 상위 디렉토리: `../document.md`
- 루트 기준: `/document.md` (권장하지 않음)

## 🚫 금지 사항

1. **중복 문서**: 동일한 내용의 문서 중복 생성 금지
2. **불명확한 제목**: 목적이 불명확한 제목 사용 금지
3. **깨진 링크**: 존재하지 않는 파일 링크 금지
4. **미완성 문서**: 완성되지 않은 문서 커밋 금지 (draft 상태 표시)
